# Zendo デザインシステム

UI実装時は必ず本ドキュメントに従う。モック議論の最終合意（2026-07-15）を集約したもの。

## 0. 原則 — 機能美

1. **色は意味にだけ使う**。装飾のための色は存在しない。「色が付いている＝何か意味がある」を全画面で成立させる
2. チェックボックス・入力欄・ナビ・罫線など**普遍的なUIは無彩色**（形が機能を説明している）
3. 色だけで意味を伝えない。**チップには必ずテキスト**（色覚多様性配慮）
4. 装飾ラベル（"TODAY"等の英字見出し）・カードの箱と影は使わない。区切りは罫線、区画は生成り面
5. 白地ベース。ステータス色が何色乗っても飽和しない

## 2. 部品ルール

| 部品 | ルール |
|---|---|
| タスク行 | 罫線(`border-b`)区切り。左: チェック円(無彩色枠24px、完了時のみ tokiwa 塗り)、中央: タイトル＋メタ行、右: 「明日へ」(nibi小文字ボタン) |
| メタ行（タイトル直下） | **先頭に期限、その後にチップ**。期限表記: 当日=時刻のみ（時刻指定なしなら非表示）/ 過去・未来=「M月D日」＋時刻あれば併記。超過は beni |
| チップ | ピル型・淡色地＋濃色文字・テキスト必須。期限超過=beni / 習慣・着手中=asagi |
| 追加操作 | **スマホ=右下FAB（mikan丸）→タップで下部に入力欄**、**PC=リスト末尾に常設入力欄**（＋記号mikan、破線下罫、"Enterで追加"）。TodayとInboxで同一デザイン・同一位置 |
| グローバルメニュー | 左スライドドロワー。スマホ=左端スワイプで開く（ボタンなし）/ PC=左上☰ボタン。項目: Inbox(バッジ)・Today・Projects(＋**直近更新4プロジェクト**をサブ表示)・Habits・Notes・Settings。アクティブ項目は mikan 左帯＋淡色地 |
| ナビゲーション | スマホ: 下部固定ナビ(Inbox/Today/Projects/Habits、kinari地、アクティブはmikan下線)。PC: 上部バー(☰＋Zendo、kinari地)のみでナビはドロワーに集約 |
| 完了フィードバック | 完了行は**即座にリストから消す**→下部トースト「◯◯を完了しました／取り消す」(sumi地・mikanアクション、5秒)。リスト末尾に「✓ 完了済み n件 ▸」折りたたみ行（タップで展開、当日完了分を表示、チェックタップで取り消し） |
| ボタン | 主要=mikan塗り白文字 / 汎用=無彩色枠。破壊的操作のみ beni |

## 3. 画面仕様（実装済み分）

### Today
- 見出し: 「M月D日（曜）」＋右に「のこり n件」(nibi)。装飾ラベルなし
- リスト: 期限超過＋今日のToDo（習慣インスタンス含む）
- 「今日の習慣」= kinari地の角丸区画。候補行＋「＋ 追加」(mikan文字ボタン)
- データ: `GET /api/today` → `{ date, todos, habitCandidates, done }`（doneは当日JSTに完了したもの）

### Inbox
- 見出し: 「Inbox」＋「未仕分け n件」
- 仕分けは**「今日」「明日」の2択のみ**（無彩色枠ボタン）。期限なしToDo化・Project化は行タップの詳細画面に移す(未実装)
- 入力欄はTodayと同一（スマホ=FAB / PC=リスト末尾常設）

## 4. 将来画面への申し送り

- 期限なしToDoの受け皿はProjects画面（プロジェクト外の単独ToDo一覧を含む）
- ステータス色を増やすときは asagi 系統に追加し、色相を増やさない
- ドロワーの「直近使用プロジェクト」= `kind=project` を `updated_at` 降順で4件

## 5. 実装・検証状況

本デザインは Today/Inbox/AppShell に適用済み。テスト34件・ビルド通過。**全項目ブラウザ検証済み（2026-07-16、PC/モバイル両ビューポート＋ダークモード）**。

検証済みの挙動:
- ドロワー開閉＋直近プロジェクト4件 / 完了→トースト→取り消し / 完了済み折りたたみ
- 期限表記（先頭・当日は時刻のみ・超過は紅で日付+時刻）/ Inbox2択仕分け / PC・モバイル要素の出し分け(`md:`)
- 常設入力欄のEnter追加（`onKeyDown`+`isComposing`ガード。IME変換確定Enterでは登録されず、通常Enterで登録）
- モバイル: FAB→入力シート（追加後もシート維持で連続入力）/ 下部ナビ / 左端スワイプでドロワー
- ダークモード: 全トークンが暗地パレットに切替（背景#171512・文字#EDE9E1・mikan/asagi/beniは明度調整）
- 「明日へ」（先送り）: 翌日へ移動＋`postponed_count`+1

補足: 検証環境（自動化ブラウザ）はスクリーンショットと合成キーイベントの配信が不安定なため、`read_page`＋computed値＋DB直接確認で検証した。Enter追加は実キーボード相当の`keydown`発火で確認済み。
