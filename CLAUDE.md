@AGENTS.md

# コミュニケーション
- 日本語で応答する（コード・変数名は英語）
- 簡潔に回答し、自明な説明は省略する
- 複雑なタスクでは実装前に計画を提示し、承認後に着手する


# コードスタイル
- 関数型アプローチを優先し、副作用を最小化する
- 厳密な型付け（anyは使わずunknownを使う）
- エラーは握りつぶさず、意味のあるメッセージ付きで処理する
- 1関数に処理を詰め込まず、小さく割る
- コメントは「なぜ」だけ残す。「何をするか」の説明・docstringは書かない
- styled-componentsを使わない（Tailwindのみ）
- divのネストを3階層以上にしない
- インラインスタイルを書かない
- !importantを使わない


# 仕様とテスト
- 仕様が曖昧なまま実装に進まない。先に質問する
- テストの追加・変更は提案にとどめ、承認後に書く
- 実装とテストが食い違う場合、テストを直さず報告する


# Git規約
- Conventional Commits形式、本文は日本語（例: feat: ユーザー認証にOAuth2を追加）
- 確認なしに自動コミット・自動pushしない


# 禁止事項
- README・ドキュメントを勝手に生成・変更しない
- テストコードを確認なしに削除・コメントアウトしない
- 既存の動作するコードを理由なくリファクタリングしない
- 依頼されていない共通化・抽象化を入れない
